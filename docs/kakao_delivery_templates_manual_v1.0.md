## 3. 템플릿별 신청 내용

> **입력 규칙**  
> - 템플릿 코드: 영문 대문자·숫자·언더스코어만 허용  
> - 메시지 본문: 코드에서 실제 발송되는 텍스트와 **한 글자도 다르면 안 됨** (승인 후 수정 불가)  
> - 변수: `#{변수명}` 형식 — 변수 포함 시 비즈센터에서 변수 목록 별도 등록 필요  
> - 버튼 없이 순수 텍스트 메시지로 신청

> **참고**: 코드에서 `order['request_title'][:30]`으로 30자 절삭, `{body.amount:,}`으로 쉼표 포함 출력

## 4. 신청 단계별 화면 안내

### Step 1. 템플릿 추가 클릭
```
알림톡 관리 → 템플릿 관리 → [+ 템플릿 추가]
```

### Step 2. 기본 정보 입력
```
채널 선택:  [AEOlab 채널 선택 ▼]  ← 기존 승인된 채널 그대로
템플릿 코드: AEOLAB_DELIVERY_01   ← §3에서 확인
템플릿 이름: 대행 서비스 접수 완료
카테고리:    구매/예약 확인        ← §3 표에서 확인
```

### Step 3. 메시지 본문 입력
```
§3의 [메시지 본문] 텍스트를 그대로 복사·붙여넣기
(공백, 줄바꿈 하나라도 다르면 실제 발송 시 불일치로 실패)
```

### Step 4. 변수 등록
```
본문에 #{변수명} 패턴 입력 시 자동 감지됨
변수 유형: '텍스트' 선택 (모든 변수 동일)
```

### Step 5. 검수 요청
```
[검수 요청] 클릭 → 영업일 기준 1~3일 소요
승인 결과: 이메일 + 비즈센터 알림
```

### Step 6. 승인 확인
```
템플릿 관리 목록 → 상태 '승인' 확인
```

## 5. 승인 후 환경변수 설정

4종 모두 승인 후 서버 `.env` 에 추가:

```bash
# 대행 서비스 알림톡 4종
KAKAO_TEMPLATE_DELIVERY_RECEIVED=AEOLAB_DELIVERY_01
KAKAO_TEMPLATE_DELIVERY_IN_PROGRESS=AEOLAB_DELIVERY_02
KAKAO_TEMPLATE_DELIVERY_COMPLETED=AEOLAB_DELIVERY_03
KAKAO_TEMPLATE_DELIVERY_FOLLOWUP=AEOLAB_DELIVERY_04   # Sprint 3 스케줄러 구현 후
```

`.env` 수정 후 PM2 재시작:
```bash
ssh root@115.68.231.57
cd /var/www/aeolab
# .env 파일 수정
pm2 restart aeolab-backend
pm2 logs aeolab-backend --lines 30 --nostream  # 오류 확인
```

## 6. 발송 테스트 절차

### 6.1 DELIVERY_01 테스트 (결제 완료 알림)
1. 대시보드 → 대행 서비스 → 패키지 선택 → 토스 결제 (test_ 키로 테스트 결제)
2. 결제 완료 후 `payment-confirm` 페이지에서 서버 확인 자동 진행
3. 본인 휴대폰에 알림톡 수신 확인

### 6.2 DELIVERY_02·03 테스트 (운영자 발송)
1. `/admin/delivery` 접속
2. 테스트 의뢰 선택 → [진행 시작] 클릭 → 알림톡 수신 확인 (DELIVERY_02)
3. [완료 처리] 클릭 → 알림톡 수신 확인 (DELIVERY_03)

### 6.3 알림 미수신 시 점검 순서
```
1. profiles.phone 컬럼에 전화번호 등록 여부 확인
   (대시보드 → 설정 → 전화번호 입력)

2. 환경변수 등록 여부 확인
   ssh root@115.68.231.57 "grep KAKAO_TEMPLATE_DELIVERY /var/www/aeolab/.env"

3. PM2 에러 로그 확인
   pm2 logs aeolab-backend --lines 60 --nostream | grep "delivery"

4. NHN Cloud 콘솔 발송 내역 확인
   console.nhncloud.com → KakaoTalk Bizmessage → 발송 조회
```

## 7. 거절 시 재신청 가이드

카카오 심사에서 거절 사유가 나오는 주요 패턴:

| 거절 사유 | 수정 방법 |
|---------|---------|
| 광고성 문구 포함 | `[AEOlab]` 외 광고·할인·이벤트 문구 제거 |
| 변수 범위 불명확 | 변수 설명란에 "최대 N자" 명시 |
| URL 형식 오류 | `https://` 포함 전체 URL 입력 |
| 카테고리 불일치 | "알림/안내" 또는 "구매/예약 확인"으로 변경 |
