# AEOlab 홈페이지 UX/UI 점검 보고서 대조 분석

> 작성일: 2026-04-26  
> 기준: ChatGPT 생성 「AEOLAB 홈페이지 UX/UI 구조 점검 보고서」vs 현재 aeolab.co.kr (v5.4 기준)

---

## 1. 핵심 결론

ChatGPT 보고서는 **구버전 홈페이지(회사 소개형 구조)** 를 기반으로 작성됐습니다.  
2026-04-23 홈페이지 개선 v1.0 이후 구조가 전면 교체되어, 보고서 지적의 약 **70%는 이미 해소**된 상태입니다.

---

## 2. ChatGPT가 잘못 알고 있는 것 (사실과 다른 항목)

### 2-1. 현재 홈페이지 구조가 "회사 소개형"이라는 전제

보고서가 분석한 구조:
```
서비스 소개 → 콘텐츠 소개 → 문의
```

실제 현재 구조 (`app/page.tsx` 기준):
```
Hero (헤드라인 + 업종 선택 + CTA)
→ 이용 방법 3단계 (HowItWorksSection)
→ 왜 AI에 안 나오나 (WhyNotShownSection)
→ AEO vs 전통 마케팅 비교 (AEOvsTraditionalSection)
→ 가격 앵커 (네이버 광고 vs AEOlab)
→ 후기 (Testimonials)
→ FAQ
→ 최종 CTA
```

→ 전형적인 **전환 중심 랜딩페이지 구조**로 이미 재설계 완료.  
"회사 소개형"이라는 보고서의 전제 자체가 현 사이트와 맞지 않음.

---

### 2-2. CTA가 "문의" 하나뿐이라는 지적

보고서: *"'문의' 외 명확한 유도 버튼 없음"*

실제 현황:
- 헤더 상단 고정: **"무료 진단 시작"** 버튼 (sticky)
- Hero 섹션: **"내 가게 AI 노출 확인하기"** (파란색 메인 CTA)
- 업종 타일: 클릭 시 `/trial`로 바로 이동
- 가격 앵커 섹션: **"요금제 보기 →"** 버튼
- 최종 CTA 섹션: **"무료로 내 가게 진단받기"** (전체 너비 버튼)
- 모바일: **MobileFloatingCTA** (하단 고정 플로팅 버튼, 스크롤 600px 이후 표시)

→ 총 5개 이상의 CTA가 이미 배치되어 있음.

---

### 2-3. 서비스 대상이 전혀 불명확하다는 지적

보고서: *"누구를 위한 서비스인지 명확하지 않음"*

실제 현황:
- Hero 헤드라인: "네이버·ChatGPT가 우리 동네에서 **먼저 추천하는 가게**"
- 서브카피: "**리뷰 100개 쌓아도 AI엔 안 나옵니다**" (자영업자 공감 문구)
- 업종 타일: 음식점·카페·미용·의료 등 25개 소상공인 업종 명시
- 헤더: "네이버·ChatGPT·Google AI 노출 관리"

→ "소상공인"이라는 단어가 직접 나오진 않지만,  
   업종 타일·헤드라인으로 타겟이 충분히 명시됨.

---

### 2-4. 문제 해결 구조가 없다는 지적

보고서: *"기술 설명 중심, 사용자 문제 → 해결 → 결과 흐름이 없음"*

실제 현황 (블록 순서):
1. **문제 인식** → `WhyNotShownSection`: "왜 AI에 안 나오나" (5가지 원인 설명)
2. **해결 방법** → `HowItWorksSection`: "3단계로 확인하는 법"
3. **결과 비교** → `AEOvsTraditionalSection`: "AEO vs 전통 마케팅"
4. **비용 설득** → 가격 앵커: "네이버 광고 월 90만원 vs AEOlab 9,900원"

→ 문제 → 해결 → 결과 흐름이 이미 구현되어 있음.

---

### 2-5. 정보 계층이 혼재한다는 지적

보고서: *"서비스 / 콘텐츠 / 기술이 동일 레벨"*

실제 현황:
- 구버전의 "콘텐츠 소개" 섹션은 완전히 제거됨
- 현재는 전환 퍼널 단계(인식→이해→비교→행동)를 따르는 단일 흐름
- `/demo` 페이지로 콘텐츠 분리됨

→ 계층 혼재 문제 해소 완료.

## 4. 아직 해소되지 않은 항목 (실제 남은 과제)

### 4-1. 후기(Testimonials) — 데이터 문제, 코드 완성됨

- 컴포넌트(`Testimonials.tsx`)와 데이터 파일(`lib/testimonials.ts`)은 이미 구현됨
- `isPlaceholder: true` 상태에서는 섹션 자체가 자동으로 숨겨짐
- **실베타 후기 1~3개 확보 후** `quote` 교체 + `isPlaceholder: false`로만 변경하면 즉시 표시
- **사용자가 직접 해야 하는 작업** (코드 작업 불필요)

### 4-2. 섹션 내 중간 CTA — 코드 작업 가능

현재 CTA 공백 구간:
- `HowItWorksSection` 하단: CTA 없음
- `WhyNotShownSection` 하단: CTA 없음
- `AEOvsTraditionalSection` 하단: CTA 없음

→ 각 섹션 하단에 "내 가게도 확인해보기 →" 링크 버튼 추가 시 이탈 감소 기대

### 4-3. 성과 데이터 (before/after) — 데이터 없음

- 실제 사용자의 AI 노출 개선 사례 수치가 없음
- 구독자 확보 후 케이스 스터디 형태로 추가 가능
- 단기 대안: DiagnosisCounter 수치가 충분히 쌓이면 "OOO개 가게 진단 완료" 강조

### 4-4. "소상공인" 타겟 명시 강화 — 코드 작업 가능

- Hero 서브카피 또는 뱃지에 "소상공인 전용" 문구 1줄 추가
- 현재는 업종 타일로 암시적 표현 → 명시적 표현으로 전환 가능

## 5. 보고서 권장 구조 vs 현재 구조 비교

| 보고서 권장 구조 | 현재 구조 | 일치 여부 |
|---------------|---------|---------|
| ① Hero (대상+문제 정의) | ✅ Hero + HeroHeadline + HeroIndustryTiles | ✅ |
| ② 문제 정의 (공감 유도) | ✅ WhyNotShownSection | ✅ |
| ③ 해결 방법 | ✅ HowItWorksSection | ✅ |
| ④ 서비스 소개 (결과 중심) | ✅ AEOvsTraditionalSection | ✅ |
| ⑤ 사례/결과/후기 | ⚠️ DiagnosisCounter만, Testimonials는 placeholder | 부분 |
| ⑥ 차별성 비교 | ✅ AEOvsTraditionalSection + 가격 앵커 | ✅ |
| ⑦ CTA (각 섹션 반복) | ⚠️ Hero·가격앵커·최종만, 중간 섹션 CTA 없음 | 부분 |
