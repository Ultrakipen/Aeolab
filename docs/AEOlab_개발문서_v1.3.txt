# AEOlab CLAUDE.md

## 프로젝트 개요
- 서비스: 소상공인 AI 검색 가시성 진단 및 성장 플랫폼
- 포지셔닝: "AI가 내 가게를 어떻게 설명하는지 진단·개선"
- 기획서: v7.2 (최신) / 개발문서: v1.3 (최신)

## 핵심 아키텍처
- 데이터 수집: 방법3(네이버 API) 우선 + 방법1(Playwright) 보조
- AI 분석: Gemini Flash (100회 샘플링) + Claude Sonnet (가이드 생성)
- 프레임워크: Next.js 14 App Router + Python FastAPI + Supabase

## 개발 원칙
- Phase 0~1: 방법3 API만 사용 (Playwright는 Phase 2부터)
- 네이버 플레이스 방문자 리뷰: 공식 API 없음 --> Playwright 제한적 사용
- 블로그 리뷰: 네이버 검색 API (블로그) 사용 (합법)
- 경쟁사 수집: 지역 검색 API로 목록 --> Playwright로 키워드 리뷰 (주 1회 한정)

05  Supabase 데이터베이스 스키마

-- 사업장 테이블
CREATE TABLE businesses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  name TEXT NOT NULL,
  region TEXT NOT NULL,
  industry TEXT NOT NULL,
  place_id TEXT,              -- 네이버 플레이스 ID
  open_period TEXT,           -- 운영 기간
  created_at TIMESTAMPTZ DEFAULT now()
);

-- AI 스캔 결과
CREATE TABLE scan_results (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  business_id UUID REFERENCES businesses(id),
  scan_type TEXT,             -- 'ai_sampling' | 'briefing_gap' | 'competitor'
  scores JSONB,               -- {visibility, review, engagement, differentiation}
  platform_scores JSONB,      -- {naver, chatgpt, perplexity, grok, claude}
  gap_keywords JSONB,         -- 미보유 키워드 목록
  raw_data JSONB,             -- 원시 수집 데이터
  scanned_at TIMESTAMPTZ DEFAULT now()
);

-- 경쟁사 정보 (캐시)
CREATE TABLE competitors (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  business_id UUID REFERENCES businesses(id),
  place_id TEXT,
  name TEXT,
  distance_m INTEGER,
  scores JSONB,
  top_keywords TEXT[],
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- 리뷰 데이터 (블로그 API + Playwright)
CREATE TABLE reviews (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  business_id UUID REFERENCES businesses(id),
  source TEXT,                -- 'naver_blog' | 'place_visitor' | 'kakao'
  content TEXT,
  keywords TEXT[],            -- 추출된 키워드
  categories JSONB,           -- {access, party, ambience, companion, menu, ops}
  published_at TIMESTAMPTZ,
  collected_at TIMESTAMPTZ DEFAULT now()
);

-- 주간 알림 로그
CREATE TABLE notification_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  business_id UUID REFERENCES businesses(id),
  type TEXT,                  -- 'score_change' | 'competitor' | 'market' | 'todo'
  content JSONB,
  sent_at TIMESTAMPTZ DEFAULT now()
);

 모든 사용자 노출 텍스트는 한국어. 변수명·함수명은 영어
 Next.js: Server Components 기본, 클라이언트 인터랙션은 'use client'
 FastAPI: 비동기(async/await) 기본, Pydantic v2 모델 사용
 에러 처리: 모든 외부 API 호출에 try-except, 사용자 친화적 메시지 반환
 환경 변수: python-dotenv / Next.js .env.local, 코드에 직접 삽입 금지
 Playwright: 요청 간 최소 2초 딜레이, 동일 URL 24시간 캐싱

7.3  테스트 우선 개발 대상

 네이버 블로그 API: 다양한 가게명으로 30개 리뷰 수집 성공 여부
 Claude API 키워드 추출: 6개 카테고리 정확도 (업종별 10개 샘플)
 점수 계산 로직: 체크리스트 변경 --> 실시간 점수 반영
 경쟁사 반경 필터: 지역 검색 API 반경 500m 내 필터링 정확도
