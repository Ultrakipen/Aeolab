# AEOlab 데이터 전략 — AI 노출 기준 데이터 DB

> 작성일: 2026-04-03  
> 목적: AEOlab이 수집하는 데이터의 성격, 구조, 활용 계획 정리

---

## 1. 이 데이터는 무엇인가

AEOlab이 수집하는 데이터는 **가게의 일반 정보가 아닙니다.**

| 구분 | 내용 | 예시 |
|------|------|------|
| 일반 가게 정보 (네이버 플레이스, 카카오맵) | 가게가 존재하는 정보 | 상호명, 주소, 전화번호, 메뉴, 리뷰 |
| **AEOlab 데이터** | 가게가 AI에 노출되는 기준과 현황 | AI 노출 점수, 노출되는 플랫폼, 부족한 키워드, 경쟁 대비 격차 |

**핵심 질문:** "이 업종/지역 가게들이 AI 검색에 얼마나, 어떻게 노출되는가?"

이 데이터를 체계적으로 수집하는 서비스는 현재 한국에 없습니다.  
네이버·카카오·구글도 소상공인에게 이 정보를 직접 제공하지 않습니다.

---

## 2. 데이터 수집 구조

스캔 한 번 = "이 가게가 지금 AI 검색에 얼마나 노출되는가" 측정

```
사용자가 스캔 실행
        ↓
8개 AI 플랫폼에 쿼리 (Gemini, ChatGPT, Perplexity, Grok, Naver, Claude, Google, Kakao)
        ↓
노출 여부, 점수, 키워드 측정
        ↓
3개 테이블에 동시 저장
```

### 저장 테이블 구조

```
scan_results          — 구독자 가게 전체 스캔 결과 (개인정보 포함)
trial_scans           — 비로그인 체험 스캔 결과
scan_analytics        — 익명화 통계 (구독자 + 체험, 개인정보 없음)
competitor_snapshots  — 경쟁사 AI 노출 현황 (익명화, 개인정보 없음)
```

---

## 3. 테이블별 저장 항목

### scan_analytics (구독자 + 체험 스캔마다 자동 저장)

가게의 AI 노출 현황을 익명화해서 저장. 업종/지역별 벤치마크 DB의 핵심.

| 컬럼 | 의미 |
|------|------|
| scan_type | 'full'(구독자) / 'trial'(체험) |
| category | 업종 (카페, 음식점, 미용실...) |
| region | 지역 (서울 강남, 부산 해운대...) |
| track1_score | 네이버 AI 채널 노출 점수 (0~100) |
| track2_score | 글로벌 AI 채널 노출 점수 (0~100) |
| unified_score | 업종별 가중 통합 점수 |
| growth_stage | 노출 단계 (survival/stable/growth/dominant) |
| top_missing_keywords | AI가 이 가게를 찾을 때 없는 키워드 |
| keyword_coverage | 업종 핵심 키워드 보유 비율 |
| platform_mentioned | 어떤 AI가 언급했는지 (JSONB) |
| competitor_count | 등록된 경쟁사 수 |
| competitor_mentioned_count | AI에 노출된 경쟁사 수 |
| competitor_avg_score | 경쟁사 평균 점수 |
| my_vs_competitor_gap | 내 점수 - 경쟁사 평균 (양수=우위) |

**개인정보 없음:** business_id, user_id, 사업장명 미포함

### competitor_snapshots (경쟁사 스캔마다 자동 저장)

경쟁사를 독립 사업장으로 저장. 업종/지역별 AI 노출 분포 DB.

| 컬럼 | 의미 |
|------|------|
| category | 업종 |
| region | 지역 |
| mentioned | AI에 노출됐는지 여부 |
| score | AI 노출 점수 (45=노출, 15=미노출) |
| track1_score | 네이버 AI 채널 점수 |
| track2_score | 글로벌 AI 채널 점수 |
| unified_score | 통합 점수 |
| growth_stage | 노출 단계 |
| scanned_at | 측정 시각 |

**개인정보 없음:** competitor_id, 상호명 미포함

## 4. 데이터 축적 원리 (플라이휠)

```
구독자가 스캔할수록
        ↓
scan_analytics + competitor_snapshots에 행 추가 (비용 없음)
        ↓
업종×지역 조합 데이터 밀도 증가
        ↓
벤치마크 정확도 향상 → 서비스 품질 향상
        ↓
구독자 증가 → 데이터 더 쌓임
```

구독자 10명 → 강남 카페 데이터 시작  
구독자 100명 → 주요 업종×지역 커버리지 확보  
구독자 500명 → 전국 단위 의미 있는 통계 가능

## 5. 데이터 활용 로드맵

### 단계 1: 서비스 품질 향상 (즉시 ~ 구독자 100명)

- **벤치마크 고도화**: "강남 카페 평균 AI 노출 점수: 42점, 당신: 38점 → 하위 40%"
  - 현재: 구독자끼리만 비교 (데이터 부족 시 의미 없음)
  - 개선: scan_analytics 기반 실제 분포 데이터로 비교
- **가이드 개선**: 실제 노출되는 가게들의 공통 키워드 → 추천 키워드에 반영
- **랜딩 페이지 신뢰도**: "서울 카페 중 AI 브리핑 노출 23%만" 같은 실제 수치 제공

### 단계 2: 신규 B2B 서비스 (구독자 100명 이후)

| 대상 | 서비스 내용 | 단가 |
|------|------------|------|
| 프랜차이즈 본사 | 전국 가맹점 AI 노출 현황 대시보드 | 월 50~100만원 |
| 창업 컨설팅사 | 지역×업종 경쟁 강도 API | 건당 3~5만원 |
| 부동산 중개 | 상권별 AI 노출 경쟁 밀도 | 건당 3~5만원 |

### 단계 3: 공개 인덱스 리포트 (구독자 500명 이후)

**"한국 소상공인 AI 검색 가시성 인덱스"** 분기별 발행

- 업종별/지역별 AI 노출 현황 통계
- 성장 단계 분포 (생존기/안정기/성장기/지배기 비율)
- 플랫폼별 노출률 트렌드

효과:
- SEO 유입: "강남 카페 AI 검색 노출" 검색어
- PR: 언론 보도 소재
- 신뢰도: 데이터 기반 서비스 입증

### 단계 4: B2G (구독자 500명 이후)

- 소진공, 지자체에 소상공인 디지털 전환 현황 데이터 납품
- 건당 수백~수천만원 규모
