## 4. 실현 가능성 검토

### ⚠️ 실현 어렵거나 리스크가 높은 항목

#### 4-1. Playwright 기반 스캐너 (네이버·Google·Zeta)
- **리스크**: 서비스 약관 위반 가능성, 차단 위험
- **기술적 리스크**: 구독자 증가 시 Playwright 동시 실행 → 서버 RAM 부족 (인스턴스당 300~500MB)
- **현실**: 네이버 AI 브리핑은 공개 페이지라 그레이존. Google AI Overview는 봇 탐지 정교함
- **권장**: 차단 대비 fallback 로직 필수, 큐 방식 순차 처리 (현재 코드에 반영됨)

#### 4-2. 뤼튼(Zeta) 스캐너
- **리스크**: wrtn.ai는 회원 로그인 필요. Playwright 세션 유지 어려움, 차단 가능성 높음
- **현실**: 장기 운영 불안정. 뤼튼 API 공식 제공 시 전환 필요
- **권장**: 일단 구현하되 "베타" 표기, 불안정 시 제외 옵션

#### 4-3. Grok AI 스캐너
- **리스크**: X(Twitter) 계정 필요, API 비용 상대적으로 높음 ($5~15/1M)
- **현실**: 한국 소상공인 대상 Grok 점유율 낮음 (153만 MAU)
- **권장**: 스캔 결과에서 중요도 낮게 표시, 비용 대비 효과 재검토

#### 4-4. 100회 샘플링 정확도 주장
- **리스크**: AI 응답은 매번 다름. 100회 샘플링이 "정확한 노출 확률"임을 증명하기 어려움
- **현실**: 통계적으로 의미 있는 수준이나, 소상공인에게 과학적 근거 설명 필요
- **권장**: "100회 측정 기반 노출 가능성 %" 대신 "100회 중 N회 언급됨"으로 직관적 표현

#### 4-5. 카카오 알림톡 5종 템플릿
- **리스크**: 심사 승인 3~5 영업일 → 반려 가능성 존재
- **현실**: 특히 마케팅성 내용은 심사 기준 엄격
- **권장**: 정보성 알림(점수 변화, AI 인용)으로 먼저 심사, 마케팅성 분리

#### 4-6. iwinv vCPU2/RAM4GB 단일 서버
- **리스크**: Playwright 2개 동시 실행 시 OOM 위험. 구독자 20명 이상 시 일일 자동 스캔 부하
- **현실**: BEP 구독자 20명 × 8 AI × Playwright = 동시 처리 불가
- **권장**: Playwright 작업 큐 필수 (Redis + 순차 처리), 구독자 50명 이후 서버 업그레이드 검토

## 5. 추가 권장 기능

### 🔴 우선순위 높음 (BEP 달성에 직접 영향)

#### 5-1. 랜딩 페이지 전환 최적화
- **현재**: 입력 폼 → 결과 → 회원가입 유도 흐름
- **추가 필요**: 실제 샘플 결과 화면 미리보기 (가입 전 결과물 확인)
- **이유**: 소상공인은 "해보기 전에 어떻게 생기는지" 봐야 결정함

#### 5-2. 온보딩 가이드
- **현재**: 가입 후 대시보드로 바로 이동
- **추가 필요**: 3단계 온보딩 (사업장 등록 → 첫 스캔 → 결과 해석)
- **이유**: 소상공인은 기술 친숙도 낮음. 첫 경험이 구독 유지 핵심

#### 5-3. 결과 공유 기능
- **현재**: 결과는 로그인 후 대시보드에서만 확인
- **추가 필요**: Before/After 카드 카카오톡·인스타그램 공유 버튼
- **이유**: 바이럴 마케팅 효과 + 소상공인 커뮤니티 확산

#### 5-4. 네이버 플레이스 연동 강화
- **현재**: naver_place_id로 통계 파싱 (리뷰 수, 평점)
- **추가 필요**: 플레이스 키워드 태그 분석, 포토리뷰 수 추적
- **이유**: 소상공인의 90%가 네이버 플레이스 사용 → 가장 친숙한 지표

### 🟡 우선순위 중간 (구독 유지에 영향)

#### 5-5. 경쟁사 자동 탐색
- **현재**: 경쟁사 수동 입력
- **추가 필요**: 업종·지역 기반 경쟁사 자동 추천 (네이버 플레이스 상위 5개)
- **이유**: 소상공인이 경쟁사 이름을 모르는 경우 많음

#### 5-6. 개선 조치 완료 체크리스트
- **현재**: AI 가이드 → 항목 목록만 제공
- **추가 필요**: 각 항목에 "완료" 체크 → 다음 스캔 시 점수 변화 예측치 표시
- **이유**: 행동 촉진 + 효과 체감 → 구독 유지

#### 5-7. 업종별 벤치마크
- **현재**: 경쟁사와 1:1 비교
- **추가 필요**: "우리 업종 평균 점수 vs 내 점수" 비교
- **이유**: "내가 평균보다 낮다"는 자극이 유료 전환에 효과적

#### 5-8. 구글 비즈니스 프로필 연동
- **현재**: Schema JSON-LD 생성만
- **추가 필요**: Google Business Profile API 연동 → 리뷰·정보 자동 동기화
- **이유**: Google AI Overview 노출에 GBP 정보가 직결됨

### 🟢 우선순위 낮음 (장기 검토)

#### 5-9. 멀티 사업장 대시보드
- 한 계정에 여러 사업장 → 통합 뷰 (현재는 1사업장 중심 UI)
- 프랜차이즈·다점포 사업자 대상

#### 5-10. AI 답변 내 위치 분석
- 언급 여부(O/X)를 넘어 → 답변의 몇 번째 문장에서 언급되는지
- "상위 노출" 개념의 AEO 버전

#### 5-11. 소상공인 커뮤니티 연동
- 당근마켓 비즈프로필, 카카오 채널과 연동
- 지역 기반 소상공인 네트워크 활용

## 6. 요금제 전략 재검토

### 현재 구조 문제점
| 항목 | 현재 | 문제 |
|------|------|------|
| Basic 9,900원 | 수동 10회 + 자동 월 1회 | 소상공인이 "월 1회"로 가치를 느끼기 어려움 |
| 창업 패키지 39,900원/3개월 | Pro와 가격 거의 같은데 기능 적음 | 구매 매력 낮음 |
| Biz 79,900원 | Pro의 2.7배 가격 | 팀 기능 외 차별점 불명확 |

### 권장 방향
1. **Basic** → "월 4회 자동 스캔 (주 1회)"으로 상향 검토 → 주간 알림과 연동 시너지
2. **창업 패키지** → 3개월 한정 + 창업 분석 리포트 강조 → 가격을 29,900원/3개월로 낮춰 입문 유도
3. **Pro** → 무제한 + 주 1회 자동은 유지하되, **리셀러 권한** 추가 (마케터가 고객사 여러 개 관리)
4. **무료 체험** → 결과 확인 후 "지금 가입하면 이번 달 무료" 30일 무료 트라이얼로 전환율 제고
