# AEOlab 홈페이지 개선 계획
**목표:** 처음 들어온 사장님이 **60초 안에** 자기 상황을 이해하고, 다음 행동으로 넘어가게 만들기
**작성일:** 2026.04.23

---

## 🎯 이 계획의 단 하나의 원칙

사장님의 머릿속에서 일어나야 할 **3개의 질문에 각각 20초 안에 답하기**:

```
┌─────────────────────────────────────────┐
│  0~20초    "이게 나한테 뭐지?"             │
│  20~40초   "진짜야? 믿을 수 있어?"          │
│  40~60초   "그래서 지금 뭘 눌러?"          │
└─────────────────────────────────────────┘
```

지금 홈페이지는 이 3개 답이 **페이지 전체에 섞여** 있습니다. 이걸 **3개의 분명한 구역**으로 나누는 것이 전체 개선의 핵심입니다.

## 📐 새 홈페이지 구조 (한 눈에 보기)

현재 15개 블록 → **5개 블록으로 압축.**

```
┌──────────────────────────────────────┐
│ 1. 히어로 (첫 화면)                      │  ← "이게 뭐지?" 답
│    ─ 한 문장 + 업종 타일 + CTA 1개        │
├──────────────────────────────────────┤
│ 2. 30초 안에 증명 (라이브 샘플)            │  ← "진짜야?" 답
│    ─ 우리 동네 실제 스캔 결과 1개          │
├──────────────────────────────────────┤
│ 3. 이게 왜 중요한가 (짧게)                │  ← 배경 설명
│    ─ 네이버 AI 40% 확대 1문장 + 숫자 3개   │
├──────────────────────────────────────┤
│ 4. ChatGPT vs AEOlab (표 1개만)         │  ← 차별화
│    ─ 지금 있는 7줄 표 그대로              │
├──────────────────────────────────────┤
│ 5. 지금 시작하기 (최종 CTA)              │  ← "뭘 눌러?" 답
│    ─ 업종 타일 + 가격 앵커 + 버튼 1개      │
└──────────────────────────────────────┘
```

지금 홈에 있는 나머지 블록들은:
- "이런 고민 있으신가요" (😟) → **삭제**
- "ChatGPT에 물어보면 / AEOlab이 알려주는 것" → **삭제**
- "업종별 AI 검색 현실" → **삭제**
- "AEOlab이 알려주는 것" → **4번 블록에 통합**
- "이런 변화를 기대할 수 있습니다" → **2번 블록에 통합**
- "어떻게 작동하나요" → **삭제** (결과 화면에서 자연스럽게 경험)
- "기존 마케팅 vs AEOlab" → **삭제** (5번에 1줄로 축약)
- 요금제 미리보기 → **삭제** (/pricing 페이지로)

## 🔨 단계별 실행 계획 (총 3단계)

개선 전체를 **3단계**로 나눴습니다. 각 단계는 **독립적으로 배포 가능**합니다. 1단계만 해도 체감 전환이 크게 달라집니다.

### 🔴 1단계 — 히어로와 CTA 흐름만 (4~6시간)

**목표:** 첫 화면과 마지막 화면만 고쳐서 "들어와서 버튼 누르는 비율"을 높이기

#### 1-1. 히어로 영역 재설계

**지금:**
```
배너: 손님 5명 중 1명이 이미 AI로 가게를 찾습니다
제목: 경쟁 가게가 AI에 나오는 이유, ChatGPT한테 물어봐도 모릅니다
서브: 네이버·ChatGPT·Gemini가 내 가게를 알고 있는지...
체크리스트 3줄
업종 드롭다운
버튼 2개
```

**바꿀 것:**
```
제목 (크게):
  "우리 동네에서 AI가 먼저 추천하는 가게,
   누구일까요?"

서브 (작게, 1줄):
  "업종만 선택하면 30초 안에 우리 동네
   AI 검색 순위를 보여드립니다"

업종 타일 (6개 + 기타):
  [🍽️ 음식점] [☕ 카페] [💇 미용실]
  [🏥 병원]   [📚 학원] [🛍️ 쇼핑몰]
  [기타 →]

하단 보조:
  · 회원가입 불필요 · 1분 · 무료
```

**왜 이게 되는가:**
- 질문형 제목 → 사장님이 "누군데?" 궁금해짐 (클릭 동기)
- 타일 6개 → 드롭다운 열기/스크롤/선택/닫기 4단계가 **탭 1번**으로
- CTA 버튼이 없어도 업종 타일 자체가 CTA 역할

#### 1-2. "결과 화면 미리보기" 버튼 이름 바꾸기

- 현재: "결과 화면 미리보기"
- 변경: **"샘플 결과로 먼저 보기 (30초)"**
- 위치: 히어로 하단 작게, 탭 부담 낮게

#### 1-3. 최종 CTA 블록 재작성

페이지 맨 아래 **"지금 시작하기"** 블록:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  네이버 광고 하루:  30,000원
  AEOlab 한 달:     9,900원
                  (첫 달 4,950원)
  
  [업종 선택 →] [내 가게 확인하기]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**왜:** 가격 비교 앵커가 있으면 사장님이 "아 싸다"를 **스스로 결론 내림**. 서비스가 말하는 것보다 10배 강함.

### 🟡 2단계 — 본문 블록 정리 (1~2일)

**목표:** 스크롤 양을 40% 줄이되 설득력은 오히려 강화

#### 2-1. 반복 블록 3개 삭제

삭제할 섹션:
- ❌ "이런 고민 있으신가요?" (😟 이모지 블록)
- ❌ "ChatGPT에 물어보면 / AEOlab이 알려주는 것" 대화형
- ❌ "업종별 AI 검색 현실" 좌우 화살표 캐러셀

**이유:** 이 3개 블록의 메시지는 전부 **7줄 비교표 하나**에 들어있습니다. 반복이 설득을 약하게 만듭니다.

#### 2-2. 샘플 카드 재설계 — 숫자에 맥락 붙이기

**지금:**
```
블루보틀 강남점   82점
커피빈 역삼점    73점
테라로사 역삼점   64점
내 카페(나)     28점
```

**바꿀 것:**
```
강남 역삼동 카페 실제 스캔 결과 (2026.04 측정)
───────────────────────────────
1위  블루보틀 강남점     82점  ⬆ 상위 5%
2위  커피빈 역삼점       73점  ⬆ 상위 15%
3위  테라로사 역삼점     64점  ⬆ 상위 30%
                                
     ── 업종 평균선 (51점) ──
                                
?위  내 카페 (예시)       28점  ⬇ 하위 20%

※ 100점 만점 · Gemini·ChatGPT 각 100회 질의
```

**바뀌는 것:**
- "상위 5%" 꼬리표로 숫자가 **체감되는 수치**가 됨
- 평균선을 시각적으로 표시해 "내가 어디쯤인가" 즉시 이해
- 측정 방식 한 줄로 신뢰도 ↑

#### 2-3. "효과 시나리오"를 1개로 압축하되 크게

현재 "강남 카페 / 마포 헤어샵 / 송파 한식당" **3개** 예시가 모두 비슷한 톤으로 나열됩니다. 이 중 **가장 구체적인 1개만** 크게 남기세요.

```
┌──────────────────────────────────┐
│ 실제 변화 사례                      │
│                                  │
│  송파구 한식당                      │
│                                  │
│  [Before]        →    [After]     │
│  AI 브리핑         ↓   단체예약      │
│  노출 0회              검색에 등장    │
│                                  │
│  한 일: 리뷰 답변에 '단체예약          │
│  가능' 키워드 추가 (2분 작업)         │
│                                  │
│  → 1주 후 변화                     │
└──────────────────────────────────┘
```

**왜 1개만 크게인가:**
3개를 나란히 두면 사장님이 "다 예쁜 얘기네" 로 흘려봅니다. 1개를 크게 두면 **내가 될 수 있는 모습**으로 봅니다.

#### 2-4. 이모지 대청소

삭제할 이모지:
- 😟😤😰 (감정 이모지 전부) — 사장님을 답답한 사람으로 시각화하는 느낌
- 🎯📋 (섹션 헤더의 중복 이모지) — 디자인 노이즈

유지할 이모지:
- ✓ ✗ (정보 기능)
- 업종 타일 아이콘 (🍽️☕💇🏥📚)

**원칙:** 이모지는 **섹션 헤더당 1개**만. 전체 페이지 10개 미만.

### 🟢 3단계 — 결과 페이지와 요금제 페이지 (2~3일)

**목표:** 진단을 본 사장님이 **즉시 행동**하게 만들기 + 요금제에서 막히지 않게 하기

#### 3-1. 결과 페이지 (/trial 결과, /demo) 최상단 재설계

진단 결과 화면의 **가장 위**에 이 박스 하나만 먼저:

```
┌────────────────────────────────────────┐
│ 👉 오늘 딱 이거 하나만 하세요               │
│                                        │
│  스마트플레이스 FAQ에                        │
│  이 문구를 추가하세요                        │
│                                        │
│  ┌──────────────────────────────┐      │
│  │ Q: 주차 가능한가요?              │      │
│  │ A: 네, 매장 앞에 무료 주차장이    │      │
│  │    있습니다. (5대 가능)          │      │
│  └──────────────────────────────┘      │
│                                        │
│  [📋 문구 복사하기]                       │
```

그 **아래**로 펼치면:
- 내 점수 / 경쟁사 순위 (접이식)
- 항목별 6가지 세부 점수 (접이식)
- 체크리스트 7개 (접이식)
- 손님이 찾는 과정 (접이식)

**왜:** 사장님은 **"오늘 뭘 해야 하지?"** 의 답 하나만 원합니다. 나머지는 **궁금한 사람만** 열면 됩니다.

#### 3-2. 요금제 페이지 상단에 "상황 질문" 추가

지금은 Basic/Pro/Biz/창업패키지 **4개를 나란히** 두고 "알아서 고르세요" 구조입니다. 이걸 **안내형**으로 바꿔야 합니다.

```
┌─────────────────────────────────────────┐
│ 나한테 맞는 플랜 찾기 (15초)                 │
├─────────────────────────────────────────┤
│                                         │
│  Q. 지금 내 상황은?                        │
│                                         │
│  ○ 지금 가게 1개를 운영하고 있어요            │
│                          → Basic 추천     │
│                                         │
│  ○ 경쟁사 변화를 매일 빠르게 알고 싶어요        │
│                          → Pro 추천       │
│                                         │
│  ○ 가게가 2개 이상이거나 대행사예요            │
│                          → Biz 추천       │
│                                         │
│  ○ 아직 창업 전이에요                       │
│                          → 창업패키지 추천   │
│                                         │
└─────────────────────────────────────────┘
```

선택하면 **해당 플랜만 크게**, 나머지는 **"다른 플랜 비교 보기 ↓"** 로 접어 두기.

**왜:** 플랜 4개를 동시에 15줄씩 비교하는 건 **결정 피로**만 만듭니다. "내 상황 → 추천 1개" 구조가 훨씬 빠릅니다.

#### 3-3. 무료 체험 입력 화면 미리보기 추가

홈 히어로 근처에 **실제 입력 화면 스크린샷 1장** 삽입:

```
   이렇게 생긴 화면에 입력하시면 됩니다 ↓
   
   [스크린샷: 가게 이름 / 업종 / 지역만 있는 1화면]
   
   ✓ 사업자번호 필요 없음
   ✓ 전화번호 필요 없음  
   ✓ 결제 정보 필요 없음
```

**왜:** 사장님이 "개인정보 받아가는 거 아닌가?" 걱정해서 버튼을 안 누릅니다. 이 의심을 **실물 이미지로 해소**해야 합니다.

## 📅 일정과 우선순위

| 단계 | 작업 내용 | 소요 시간 | 완료 후 효과 |
|---|---|---|---|
| **1단계** | 히어로 + 최종 CTA 재설계 | 4~6시간 | 첫 화면 이탈률 감소, 클릭률 상승 |
| **2단계** | 본문 블록 정리, 샘플 카드 개선 | 1~2일 | 페이지 이해도 ↑, 스크롤 완독률 ↑ |
| **3단계** | 결과 페이지, 요금제 페이지 재설계 | 2~3일 | 유료 전환율 상승 |

**권고 순서:** 1단계 → 3단계 → 2단계

**왜 이 순서인가:** 1단계는 **유입 → 진단** 전환에, 3단계는 **진단 → 결제** 전환에 직접 영향. 2단계는 페이지 품질 전반을 올리지만 즉각 전환에 덜 영향. 결제까지 가는 경로를 먼저 뚫고, 중간 품질은 나중에.

## 🚨 절대 지켜야 할 원칙 3가지

### 원칙 1. "설명" 말고 "경험"을 주세요

지금 홈페이지는 **"AEOlab이 이런 서비스입니다"** 를 4번 설명합니다. 개선 후에는 **"30초만 써 보세요, 보여드릴게요"** 로 가야 합니다. 설명은 최소화, 체험은 최대화.

### 원칙 2. 정보 덧붙이기가 아니라 **덜어내기**로

개선할 때 자연스러운 본능은 "이 정보도 넣자, 저 안내도 추가하자" 입니다. 반대입니다. **현재 홈페이지 텍스트의 40%를 지우는 것**이 이 개선의 핵심입니다. 뭔가 더 설명하고 싶어질 때마다 "이걸 빼면 이해가 안 될까?"를 물어보세요. 대부분은 빼도 됩니다.

### 원칙 3. 모바일 기준으로 설계

사장님의 70%+는 카카오톡 공유로 스마트폰으로 접속합니다. 모든 개선을 **iPhone 13 Pro 세로 화면(390×844)** 기준으로 검증하세요. 데스크톱에서 예뻐 보이는 구조가 모바일에서는 답답합니다.

## 💬 마지막 한 마디

이 개선 계획의 목표는 **콘텐츠를 더 좋게 만드는 것이 아니라, 사장님이 이 페이지를 떠날 때 머릿속에 남는 메시지를 "한 줄"로 만드는 것**입니다.

지금 이 페이지를 떠난 사장님에게 "뭐 하는 서비스였어요?" 라고 물으면 5가지 답이 나올 수 있습니다:
- "AI에 내 가게 나오게 하는 거?"
- "ChatGPT 어쩌고..."
- "네이버가 뭐 바뀐대..."
- "뭐 점수 나오는 거?"
- "...기억 안 나요"

개선 후에는 **"우리 동네에서 AI가 먼저 추천하는 가게 1위가 누군지 알려주고, 내 가게가 몇 등인지 보여줘"** 한 줄로 기억되어야 합니다. 이게 전부입니다.
