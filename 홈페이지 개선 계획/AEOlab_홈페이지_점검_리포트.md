## 🟢 총평 — 한 줄로

**콘텐츠의 양과 논리는 경쟁 제품 중 최상위권이지만, 정보가 너무 "평등하게" 나열되어 핵심 메시지가 묻히는 전형적 1세대 SaaS 랜딩 구조입니다.**
기획서 v7.1의 "사업성 검증 단계" 목표(Phase 0 → 유료 20명 BEP)와 비교하면, 지금 홈은 **"읽으면 다 납득되지만, 스크롤 중간에 이탈하는 구조"** 입니다. 메시지는 옳고 데이터도 정확한데, 타깃(AI 리터러시 낮은 소상공인)에게는 과밀합니다.

## 1. 구조적 문제 — 가장 먼저 손봐야 할 3가지

### 1-1. 히어로 영역이 "카피 + 데이터 + 업종선택 + 샘플표"를 한 번에 보여줌 — 정보 과부하

현재 첫 화면에 이 모든 것이 동시에 존재합니다:

- 상단 배너: "손님 5명 중 1명이 이미 AI로 가게를 찾습니다"
- 메인 카피: "경쟁 가게가 AI에 나오는 이유, ChatGPT한테 물어봐도 모릅니다"
- 서브카피 3줄 체크리스트
- 업종 드롭다운(14개 옵션)
- 2개의 CTA 버튼
- 그리고 **바로 아래 블루보틀/커피빈/테라로사 샘플 카드 4장**

**문제:**
소상공인이 모바일로 처음 들어왔을 때, 첫 스크린에서 **뇌가 결정해야 할 일이 5개**입니다. 헤드라인 이해 → 업종 선택 → 버튼 2개 중 선택 → 샘플 해석. 이 중 어느 하나에서 멈추면 이탈합니다.

**개선 방향:**
히어로는 **"한 질문"만 던져야 합니다.**

```
[히어로 — 3초 내에 읽히는 단 하나의 메시지]
   ChatGPT에 "우리 동네 OOO 맛집"을 물어보면
   내 가게가 나올까요?

   [업종 선택] [지금 30초 안에 확인하기 →]
```

샘플 카드(블루보틀/커피빈)는 스크롤 1페이지 아래로 내리고, 히어로에는 **"지금 몇 명의 소상공인이 확인 중"** 같은 live 시그널 1개만 남기세요.

---

### 1-2. "차별화 증명"이 동일한 논리를 4번 반복 — 논리는 맞지만 지루함

현재 페이지에는 "AEOlab vs ChatGPT" 구조의 비교가 **네 번** 등장합니다:

| 위치 | 형태 |
|---|---|
| 상단 "ChatGPT가 못 하는 것" 섹션 | 긴 설명 박스 |
| 중간 "이런 고민 있으신가요" 이후 | 대화형 말풍선 |
| 그 아래 | 7줄짜리 큰 비교표 |
| /pricing 페이지 상단 | 7개 아이콘 블록 (📊📡🔍🔔✅🏆💬) |

**문제:**
네이버에서 처음 유입된 사장님은 이 논리를 **세 번째에 받을 때 이미 "아 이건 ChatGPT 대신 쓰는 거구나"** 로 충분히 이해합니다. 그 이후의 반복은 설득이 아니라 **"왜 이렇게 우기지?"** 라는 인상을 줍니다.

**개선 방향:**
- 랜딩에서는 **비교표 1개만** 유지 (지금의 7줄짜리 표가 가장 강력함)
- 나머지 3개 블록은 **"효과 증명", "효과 예시 시나리오"** 등 지금 부족한 영역으로 교체
- 비교 논리의 핵심은 **"ChatGPT는 방법을 말해줄 뿐, AEOlab은 '내 가게에 지금 뭐가 없는지'를 집어준다"** 한 문장이면 충분

### 1-3. "이 가격을 쓸 근거"가 너무 늦게 나옴

현재 스크롤 순서:

1. 히어로
2. 샘플 카드 (강남 카페)
3. 순위 카드 (같은 정보 반복)
4. 분석 대상 AI 플랫폼
5. 업종별 AI 검색 현실
6. 이런 고민 있으신가요
7. ChatGPT vs AEOlab 대화형
8. 비교표
9. 네이버 AI 브리핑 설명
10. 효과 시나리오
11. 작동 방식
12. 무료 vs 유료 차이
13. 기존 마케팅 vs AEOlab
14. **드디어 요금제** ← 여기서야 9,900원이 나옴

**문제:**
기획서 v7.1에서 강조한 **"9,900원 = 네이버 광고 하루치 1/3"** 의 메시지가 페이지 중간에야 등장합니다. 소상공인이 그 전에 **"비싸면 어차피 못 써"** 라고 판단해서 이탈할 가능성이 매우 높습니다.

**개선 방향:**
히어로 바로 아래, 샘플 카드 직전에 **앵커 카드 1개** 추가:

```
네이버 키워드 광고 하루: 3~10만원
AEOlab 한 달:          9,900원 (첫 달 4,950원)
```

이 한 줄이 첫 스크롤에 보이면, 이후 정보를 **"내가 감당 가능한 가격의 제품"** 프레임으로 읽게 됩니다.

## 2. 시각·디자인 문제

### 2-1. 시각 위계(Visual Hierarchy)가 없음 — 모든 블록이 똑같이 중요해 보임

현재 거의 모든 섹션이 **"제목 + 부제 + 박스 나열"** 동일 패턴입니다. 결과적으로:

- 사용자의 눈이 "어디에 집중해야 할지" 단서를 못 받음
- 스크롤을 **정보량이 아니라 피로도**로 측정하게 됨
- 모바일에서는 세로로 한없이 이어지는 동일 크기 카드가 계속됨

**개선 방향 — 3단계 강약 체계로 재설계:**

| 단계 | 쓰임 | 시각 표현 |
|---|---|---|
| **L1 (핵심)** | 히어로, 가격 앵커, 최종 CTA | 풀블리드 배경색, 큰 타이포(모바일 32px+), 여백 충분 |
| **L2 (증거)** | 샘플 결과 카드, 비교표, 시나리오 | 카드 형태, 중간 타이포, 아이콘 사용 |
| **L3 (보조)** | 플랫폼 로고, FAQ, 푸터 | 텍스트 중심, 최소 공간 |

지금은 사실상 전부 L2 수준에 몰려 있습니다.

---

### 2-2. 숫자·점수의 신뢰감이 떨어지는 표기

샘플 카드에서 "블루보틀 82점 / 내 카페 28점" 이 나오는데, 이 숫자의 **근거가 보이지 않습니다.** 소상공인은 이 단계에서 두 가지 의심을 합니다:

1. "저 숫자는 어떻게 계산됐는데?"
2. "그냥 임의로 만든 홍보용 숫자 아닌가?"

**개선 방향:**
숫자 옆에 **아주 작게 측정 근거 한 줄**을 붙이세요.

```
82점  ← Gemini·ChatGPT 각 100회 질의 · 2026.04 측정
```

또는 점수를 클릭하면 **"왜 82점인가? 항목별 점수 보기"** 가 열리게. /demo 페이지에는 있지만 메인에는 없습니다.

### 2-3. 이모지 과다 사용 — 전문성이 깎임

랜딩 페이지 스크롤 내내 등장하는 이모지:
😟 😤 😰 ☕ ✂️ 🍱 📍 💡 📊 📡 🔍 🔔 ✅ 🏆 💬 📋 🎯 📈 🎉 ✓ ✗ ⚠️

**문제:**
기획서의 포지셔닝은 **"B2G·투자 유치용 전문 SaaS"** 인데, 이모지 밀도가 2020년대 초 소상공인 카페 수준입니다. 특히 "😟 😤 😰" 같은 감정 이모지는 타깃 무시 신호로 읽힐 수 있습니다(사장님을 답답한 사람으로 시각화하는 것처럼).

**개선 방향:**
- 이모지는 **섹션 헤더당 1개, 전체 페이지 10개 미만**으로 제한
- 감정 이모지(😟 😤 😰)는 전부 제거, 대신 **실제 스크린샷·일러스트**로 대체
- ✓ ✗ 마크는 유지해도 OK (정보 밀도 기능)

### 2-4. 색상 신호가 약함 — 모든 CTA가 "파랑 또는 검정"

"무료 진단" "무료 회원가입" "요금제 보기" 버튼들이 **전부 비슷한 색**으로 보입니다. 이것은 **전환 설계의 치명적 약점**입니다.

**개선 방향 — 1-2-3 CTA 위계:**

| CTA | 역할 | 색상 |
|---|---|---|
| **Primary** | "지금 무료 진단" (전환 최전선) | 브랜드 강조색 (현재의 파랑보다 더 높은 채도) |
| **Secondary** | "결과 미리보기" (비결제 탐색) | 보더만 있는 고스트 버튼 |
| **Tertiary** | "FAQ", "문의" | 링크 텍스트만 |

지금은 세 개가 거의 구분 없이 있어서, 사용자가 어느 버튼을 눌러야 하는지 불분명합니다.

### 🔴 수정 강력 권고 — 히어로 카피

**현재:** "손님 5명 중 1명이 이미 AI로 가게를 찾습니다"
**문제:** 근거 부족. 이 주장을 뒷받침할 1차 통계가 현재 없음. 소상공인 중 "진짜 그런가?" 의심하는 사장님이 **처음 4초 안에 신뢰를 잃는 카피**.

**대안 3가지:**

1. **미래형 (가장 안전):** "네이버 검색의 40%가 AI 브리핑으로 바뀝니다 — 2026년 안에"
2. **체험형:** "지금 ChatGPT에 우리 동네 OOO를 물어보면, 누가 나올까요?"
3. **대비형:** "네이버 광고 하루 3만원 vs AI 노출 최적화 월 9,900원"

기획서 v7.1의 "미래 위협 프레임"과 정확히 일치하는 쪽은 1번입니다.

## 4. 사용자 여정상 구멍 — Funnel Holes

홈 → 무료진단 → 회원가입 → 유료 전환의 흐름을 따라가 보면 **3개 구멍**이 보입니다.

### 구멍 ① 히어로에서 무료진단으로 갈 때 마찰

현재 히어로의 업종 드롭다운은 14개 옵션입니다. **모바일에서 스크롤해야 자기 업종이 나옴.** 기획서 v7.1에서 Phase 0 최우선 타깃으로 정한 **병원·한의원, 학원·교육, 법률·세무, 쇼핑몰**을 상단에 고정 노출하면 타깃 적중 사장님의 클릭이 유의미하게 오를 것입니다.

또한 드롭다운 대신 **"업종 타일 그리드 4~6개"** 가 모바일에서 훨씬 직관적입니다.

### 구멍 ② /demo 페이지에 직접 진입한 사용자가 길을 잃음

/demo는 정보가 풍부한데, **"이게 내 가게 결과가 아님"이 반복적으로 고지**됩니다. 사장님은 "그럼 내 가게 결과는 어디서?"를 계속 찾게 되는데, 페이지 하단까지 스크롤해야 "내 가게 무료 진단하기" 버튼이 나옵니다. **Floating CTA**(스크롤 중에도 고정으로 떠 있는 버튼)가 필수입니다.

### 구멍 ③ 요금제에서 "Basic 시작하기" → "회원가입"으로 가는 구간

현재 요금제 페이지의 Basic "시작하기" 버튼을 누르면 회원가입으로 갑니다. **그런데 회원가입 전에 "진단 결과"를 먼저 봤어야 결제 의향이 생기는데, 이 순서가 깨져 있습니다.**

**권고 순서:**
```
홈 → 무료진단(업종·지역 입력) → 진단결과 표시 → 결과 하단에 Basic 제안
     → 회원가입(결과 저장용) → 결제
```

지금은 요금제 페이지에서 바로 회원가입으로 가는 루트가 열려 있어, **결과를 안 본 사람이 가격부터 보고 이탈**합니다.

## 5. 가장 강한 부분 — 절대 손대지 마세요

이 요소들은 경쟁 제품이 따라올 수 없는 AEOlab의 진짜 자산입니다:

1. **"ChatGPT에 물어봐도 내 가게 데이터는 모른다"** — 이 통찰 자체가 한국 AEO 시장에서 최초 수준의 프레이밍
2. **비교표 7줄** — 기능 비교가 아니라 "사장님이 알고 싶은 것" 기준으로 쓰인 것이 탁월
3. **"복사 버튼 + 붙여넣기 문구"** — AEO 시장에서 이 UX를 하는 경쟁사 없음. 기획서의 "효과 증명 3레이어" 중 가장 강한 체감 도구
4. **네이버 크롤링 차단 사실 공개** — 이 사실을 알고 대응 방법을 가진 서비스는 국내에서 사실상 AEOlab만
5. **무료진단 회원가입 불필요** — 이탈 지점을 없앤 올바른 설계

## 6. 우선순위별 개선 로드맵

Phase 0(소상공인 30명 인터뷰) 전에 반드시 할 것을 우선순위 별로 정리했습니다.

### 🔴 이번 주 안에 (4~8시간 작업)

1. **히어로 카피 교체**: "손님 5명 중 1명" → "네이버 검색의 40%가 AI로 바뀝니다" 계열
2. **히어로 영역 단순화**: 샘플 카드 삭제, 업종 타일 4~6개 + CTA 1개만 남기기
3. **가격 앵커 삽입**: 히어로 바로 아래 "네이버 광고 하루 vs AEOlab 한 달" 1줄
