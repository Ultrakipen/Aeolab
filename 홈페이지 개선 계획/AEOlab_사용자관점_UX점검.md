# AEOlab 사용자 관점 UX 점검
**점검 기준:** "AI가 뭔지 잘 모르는 사장님이 카카오톡 링크로 들어왔을 때, 막힘없이 결과를 보고 행동할 수 있는가"
**점검일:** 2026.04.23

---

## 🔎 점검 방법

기획서·마케팅 의도는 전부 배제하고, **5가지 페르소나**로 사이트를 직접 돌아다녔습니다.

1. **카카오 사장님 (55세, 한식당 운영)** — "AI가 뭐가 좋다길래"
2. **인스타 원장님 (38세, 미용실)** — "요즘 네이버 안 된다던데"
3. **바쁜 카페 사장 (42세, 홍대)** — 모바일로 3분 이내 판단
4. **의심 많은 사장 (50세, 학원)** — "광고 아닌가? 속이는 거 아닌가?"
5. **이미 쓰고 있는 사람 (34세, 쇼핑몰)** — 기능 스펙을 정확히 알고 싶음

이 5명이 공통적으로 막힌 지점만 뽑았습니다.

---

## 🔴 구조적 문제 — 사용자가 실제로 막히는 지점

### 문제 1. 홈페이지에 "지금 당장 내가 뭘 해야 하는지" 없음

홈페이지를 위에서 아래까지 스크롤하면 이런 정보를 받습니다:

- AI 검색이 중요하다
- 경쟁사가 AI에 나오고 있다
- 무료 진단이 가능하다
- ChatGPT보다 우리가 낫다 (4번 반복)
- 네이버 AI 브리핑이 곧 확대된다
- 요금제가 있다

**그런데 정작 사장님이 궁금한 것:**

> "그래서 지금 이 화면에서 내가 뭘 눌러야 결과를 볼 수 있다는 거지?"

이 질문에 대한 답이 **상단 업종 드롭다운 하나**에만 담겨 있는데, 그게 **샘플 카드, 고민 섹션, 비교표** 등 수많은 블록에 파묻힙니다. CTA 버튼은 많은데 **"지금 첫걸음은 이거"라는 명확한 단일 경로**가 없습니다.

**사용자 체감:**
> "버튼은 많은데 어디를 눌러야 하는지 모르겠네..."

---

### 문제 2. 같은 정보가 여러 번 반복되어 "스크롤 해도 새 내용이 없는 느낌"

홈에서 스크롤을 내리면 사용자는 **"아까 봤던 내용인데?"** 를 반복합니다:

| 스크롤 위치 | 내용 | 사용자 반응 |
|---|---|---|
| 상단 | 샘플 카드 (블루보틀 82점 / 내 카페 28점) | "오, 비교해 주는구나" |
| 그 아래 | 같은 정보를 1~4위 순위 형태로 | "어? 아까 본 거 아닌가?" |
| 중간 | "ChatGPT는 못 하지만 AEOlab은..." | "알겠어요" |
| 중간+ | 같은 논리의 대화형 말풍선 | "또?" |
| 중간++ | 같은 논리의 7줄 비교표 | "...왜 자꾸?" |

**사용자 체감:**
> "설명은 많은데 정작 새로운 정보는 없네. 페이지가 길기만 하다."

스크롤 길이 자체가 **"이해가 안 됐을까 봐 4번 설명"** 처럼 느껴져 오히려 신뢰가 떨어집니다.

---

### 문제 3. 숫자의 의미를 모르겠음 — "82점이 좋은 건가?"

샘플 카드에 "블루보틀 82점 / 내 카페 28점"이 나오는데, 사장님이 겪는 혼란:

- 100점 만점인가? → 추측은 되지만 명시 없음
- 82점이 "좋은 건가"? → 기준 없음
- 28점이면 "망한 건가"? → 판단 불가
- 업종 평균은 몇 점? → 없음

**/demo 페이지에는 있습니다.** "C 62점, 수원시 음식점 평균 51점, 상위 40%" 라고 명확히 나옵니다. 그런데 **홈에는 이 맥락이 없어** 숫자가 그냥 "뭔가 낮은 숫자"로만 보입니다.

**사용자 체감:**
> "점수를 왜 주는 건지, 몇 점부터가 괜찮은 건지 모르겠네."

---

### 문제 4. 무료 진단을 누르기 전에 "뭐가 필요한지" 모름

홈의 CTA는 대부분 "무료 진단" 으로 보냅니다. 그런데 사장님이 가장 먼저 하는 생각:

> "뭐 입력해야 하지? 상호명? 주소? 사업자번호? 전화번호? 내 정보 받아가는 거 아냐?"

**홈페이지 어디에도 "입력은 1줄, 1분, 회원가입 불필요"라는 구체가 없습니다.** 문구로는 있는데, **실제 입력 화면이 어떻게 생겼는지 미리보기 이미지가 없습니다.**

특히 소상공인은 "개인정보 받아가려는 거 아냐?" 경계심이 큽니다. 이게 해소되지 않으면 버튼을 안 누릅니다.

**사용자 체감:**
> "누르면 뭘 해야 하는지 몰라서 안 누르게 돼."

---

### 문제 5. 업종 선택이 드롭다운이라 모바일에서 불편

현재 업종 선택은 **드롭다운(select box)** 으로 14개 옵션이 들어 있습니다. 모바일에서:

1. 탭하면 OS 기본 선택창이 화면을 덮음
2. 스크롤해서 자기 업종 찾아야 함
3. 내 업종이 정확히 뭐에 해당하는지 판단이 필요 ("한의원"은 "의료·건강"? "전문직"?)
4. 선택하고 닫아야 다음 버튼이 보임

**이게 전부 히어로 안에서 일어납니다.** 첫 3초의 단순함이 깨집니다.

**대안:** 업종 아이콘 타일 4~6개 + "기타" 버튼 1개
- 타일은 한 번에 보이고
- 탭 한 번으로 선택+다음 단계 이동이 동시에 일어남

---

### 문제 6. "결과 화면 미리보기" 버튼이 어디로 가는지 짐작 안 됨

히어로에 두 버튼이 있습니다:

- **"지금 무료로 내 가게 확인하기 →"** (분명함)
- **"결과 화면 미리보기"** (뭘까?)

두 번째 버튼을 누르면 `/demo` 페이지로 가서 **"가상의 수원시 왕갈비 한우마당 예시 결과"** 가 나옵니다. 이건 훌륭한 콘텐츠인데, **버튼 이름이 모호해서 안 누르게 됩니다.**

더 정확한 이름:
- "실제 결과 화면 예시 보기 (30초)"
- "어떤 화면이 나오는지 미리 보기"
- "내 가게 아닌 샘플로 먼저 보기"

**사용자 체감:**
> "미리보기? 뭐 미리 보는 건지 모르겠어서 안 눌러."

---

### 문제 7. 요금제 페이지에서 Basic/Pro/Biz/창업패키지 4개 중 선택이 너무 어려움

요금제 페이지에 플랜이 **4개 + 무료 체험** 까지 총 5개가 병렬로 나열됩니다. 각 플랜마다:

- 제목, 서브타이틀, 강조 문구, 가격, 특징 12~15줄

**플랜 1개당 15줄을 4번 읽어야 비교가 됩니다.** 모바일에서는 세로로 끝없이 이어지는 카드 5개를 읽어야 합니다.

하단에 비교표가 있긴 한데, **이미 위에서 절반은 이탈**합니다.

**사용자 체감:**
> "뭘 골라야 할지 모르겠어서 그냥 나갔어."

**개선 방향 — 3초 안에 자기 위치를 찾게 하기:**

```
┌─────────────────────────────────┐
│  나는 어떤 사장님인가요?            │
├─────────────────────────────────┤
│ ○ 1개 가게, 지금 시작         → Basic │
│ ○ 경쟁사를 매일 보고 싶다      → Pro   │
│ ○ 매장이 2개 이상             → Biz   │
│ ○ 아직 창업 준비 중          → 창업패키지 │
└─────────────────────────────────┘
```

이렇게 **상황 기반 진입**을 먼저 두고, 추천 플랜 하나만 크게, 나머지 3개는 작게 표시하세요.

### 문제 8. 무료 진단 후 "그래서 지금 뭘 해야 하지?"가 약함

/demo 페이지를 보면, 예시 결과 화면에 정보가 매우 많습니다:

- AI 노출 62점
- 네이버 채널 48점 / 글로벌 AI ??점
- 항목별 6가지 점수
- 체크리스트 7개
- 손님이 가게 찾는 과정 (3단계)
- 구독 시 추가 제공 항목

**좋은 정보인데, "이 중에서 지금 당장 뭐부터 할까?" 답이 묻힙니다.**

페이지에 "이번 주 집중할 것 1개" 같은 작은 박스가 있긴 한데, 그게 **전체 정보량의 1/20 분량**이라 눈에 안 들어옵니다.

**개선 방향:**
결과 화면 **최상단에 "오늘의 할 일 1개"** 를 가장 크게 배치하세요.

```
┌──────────────────────────────────────┐
│  👉 오늘 당장 이거 하나만 하세요          │
│                                      │
│  스마트플레이스 FAQ에                    │
│  "주차 가능" 한 줄 추가하기               │
│                                      │
│  [문구 복사하기]  예상 소요: 2분           │
└──────────────────────────────────────┘
```

그리고 **나머지 모든 점수·항목·체크리스트는 접어 두기.** "자세히 보기" 로 펼치게.

### 문제 9. "내가 한 게 효과 있는지" 확인 경로가 홈에 안 보임

구독의 핵심 가치가 "7일 후 점수 변화 자동 기록" 인데, 이게 홈에서는:

- /pricing 페이지 상단 아이콘 7개 중 1개
- /demo 페이지 하단의 표 한 줄

**홈에는 전혀 강조되지 않습니다.** 대신 홈에서는 "ChatGPT 못 함"만 반복됩니다.

사장님이 9,900원을 내는 이유는 "ChatGPT보다 낫다"가 아니라 **"내가 바꾼 게 효과 있었는지 알고 싶다"** 입니다. 이 가치가 홈에 안 보입니다.

### 문제 10. 푸터 정보와 신뢰 신호가 약함

홈 맨 아래에 사업자 정보만 있고:
- 실제 사용 중인 사장님 후기 0개
- 누적 진단 횟수 표시 없음 ("지금까지 많은 사업장이 무료 진단을 받았습니다" 문구만 있고 구체 숫자 없음)
- 언론 보도·인증·제휴 로고 없음

처음 온 사장님은 "이게 진짜 쓸 만한 서비스인가? 개인이 만든 가짜 아닐까?" 확인할 단서가 없습니다.

당장 못 넣더라도 **"Phase 0에서 인터뷰한 사장님 3명의 실명 후기"** 같은 작은 신뢰 신호라도 하나 있으면 전환이 달라집니다.

## ✏️ 실행 순서 — 이 순서대로만 고치면 됩니다

사용자 관점에서 영향이 큰 순서로 6단계만 정리했습니다.

### 1단계 — 히어로를 "단 하나의 결정"으로 (1시간)

현재: 카피 + 서브카피 + 체크리스트 + 업종 드롭다운 + 버튼 2개

변경: **"업종 타일 4~6개 → 탭 한 번에 결과 페이지"**

```
━━━━━━━━━━━━━━━━━━━━━━━━
 우리 동네에서 AI에 추천되는
 가게는 누구일까요?

 [🍽️음식점] [☕카페] [💇미용실] [🏥병원]
 [📚학원]  [🛍️쇼핑몰]  [기타 업종 →]

 · 30초 · 회원가입 없음 · 무료
━━━━━━━━━━━━━━━━━━━━━━━━
```

버튼 "결과 화면 미리보기"는 → **"샘플로 먼저 보기 (30초)"** 로 이름 변경

---

### 2단계 — 반복 블록 제거 (1시간)

홈에서 삭제해야 할 섹션:

- ❌ "이런 고민 있으신가요" (😟 이모지 4개)
- ❌ "ChatGPT에 물어보면 / AEOlab이 알려주는 것" 대화형
- ❌ "업종별 AI 검색 현실" 캐러셀 (정보가 추상적)

**남길 것:** 7줄짜리 비교표 1개만.
**비운 자리에 넣을 것:** "실제 사용 사례" 또는 "효과 변화 예시 1개 크게"

---

### 3단계 — 숫자에 맥락 붙이기 (30분)

현재: "블루보틀 82점 / 내 카페 28점"

변경: "블루보틀 **82점** _(상위 5%)_ / 내 카페 **28점** _(업종 평균 51점 이하)_ / 100점 만점"

이 한 줄의 맥락만 추가해도 숫자가 "체감되는 숫자"가 됩니다.

---

### 4단계 — 무료 진단 입력 화면 미리보기 이미지 삽입 (1시간)

히어로 근처에 **실제 입력 화면 스크린샷 1장**을 작게 붙이세요. 사장님이 버튼을 누르기 전에 "아, 이 정도만 입력하면 되는구나" 를 눈으로 확인하게.

```
[이미지: "가게 이름 한 줄 + 업종 + 지역" 만 있는 입력 화면]
      ↑ 이 화면이 나옵니다 (30초면 끝)
```

---

### 5단계 — 요금제 페이지를 "상황별 추천" 구조로 (2시간)

플랜 4개를 병렬로 보여주지 말고, **상황 질문 → 추천 플랜 1개 크게 → 다른 옵션 작게** 구조로.

```
[질문 박스]
 내 상황은?
 ① 가게 1개, 지금 시작  → Basic 추천
 ② 경쟁사 매일 추적 필요  → Pro 추천
 ③ 매장 2개 이상        → Biz 추천
 ④ 아직 창업 준비 중    → 창업패키지
```

선택 시 해당 플랜만 큰 카드로, 나머지는 "다른 플랜 비교 →" 링크로.

---

### 6단계 — 결과 화면 상단에 "오늘의 할 일 1개" 크게 (2시간)

/demo 페이지, 그리고 실제 /trial 결과 페이지의 **최상단**에:

```
┌──────────────────────────────┐
│ 👉 오늘 당장 이거 하나만           │
│                              │
│ [구체적인 할 일 1개]              │
│ [문구 복사]  예상 시간: 2분        │
└──────────────────────────────┘
```

그 아래로 점수·항목별·체크리스트는 **접이식(accordion)** 으로.
사장님은 "할 일 1개 → 2분 후 복사 완료 → 사이트 떠남"의 흐름을 겪어야 하고, 궁금한 사람만 더 펼쳐 보면 됩니다.

## ✅ 이 6단계만 하면 달라지는 것

| 지표 | 현재 | 개선 후 |
|---|---|---|
| 첫 화면에서 뭘 해야 할지 파악 | 5~8초 | 2~3초 |
| 홈 → 무료진단 클릭까지 스크롤 | 여러 번 | 0번 (히어로에서 끝) |
| 요금제 비교 시간 | 2~3분 | 30초 |
| 결과 화면 보고 행동까지 | "뭐부터 할지 고민" | 즉시 복사 |
