테스터 의견: "정보가 많지만 이해 어렵고, 정보를 찾아야 해서 놓치는 게 있다"

  진짜 원인 (검증 결과):
  1. 랜딩(1,020줄)·trial(2,222줄)·dashboard(1,312줄) 3대 페이지 모두 과적재
  2. 랜딩 ↔ demo ↔ trial 역할 중복 — 같은 정보를 3번 반복
  3. CTA 문구 9종으로 일관성 부족
  4. 페이지간 state 전달 없어 사용자가 같은 정보 재입력

  ▎ ⚠ ChatGPT 피드백의 "블로그·키워드 페이지 만들어라"는 무시 (SaaS 모델 오해)

  ---
  📋 5단계 실행 계획

  Phase 0 — 측정 먼저 (1일, 비용 0)

  데이터 없이 섹션 지우면 애먼 것 지울 위험.

  - GA 또는 Plausible 설치
  - 랜딩 → trial → signup 전환율 측정
  - 스크롤 뎁스로 이탈 지점 파악

  Phase 1 — 안전한 정리 (이번 주, 리스크 0)

  근거 없이도 확실히 지워도 되는 것들.

  - 섹션 8.5(기존 SEO 비교) 삭제 — 섹션 3과 중복
  - 섹션 1.5(업종 로테이터) 히어로 흡수 or 삭제
  - 섹션 번호 재정렬 (1·1.5·2·3·4·5·7·8.5·9 → 1~7 연속)
  - CTA 문구 2~3개로 통일 ("무료 진단 시작" / "1분 무료 체험")

  예상 효과: 9섹션 → 7섹션, 코드 가독성 ↑

  Phase 2 — 페이지 역할 분리 (1주)

  ┌─────────┬──────────────────┬────────────────────────────────────────────┐
  │ 페이지  │       역할       │                    조치                    │
  ├─────────┼──────────────────┼────────────────────────────────────────────┤
  │ /       │ Why — 왜         │ 섹션 7 축약: 스크린샷 1장 + "/demo 보기"   │
  │ (랜딩)  │ 필요한가         │ 링크                                       │
  ├─────────┼──────────────────┼────────────────────────────────────────────┤
  │ /demo   │ What — 결과 모습 │ 현 상태 유지 (풀 미리보기)                 │
  ├─────────┼──────────────────┼────────────────────────────────────────────┤
  │ /trial  │ How — 실제 진단  │ 설명 최소화, 결과 집중                     │
  └─────────┴──────────────────┴────────────────────────────────────────────┘

  - 히어로 폼 데이터를 trial로 state 전달 → 중복 입력 제거
  - demo·trial 결과 화면 중복 요소 정리

  Phase 3 — trial 페이지 분해 (2~3일, 핵심)

  2,222줄 → 3개 파일로:
  - trial/input/ — 입력 단계
  - trial/scanning/ — 진행 화면
  - trial/result/ — 결과 화면
  - 결과 우선순위 재설정 — 가장 중요한 1개만 초기 노출, 나머지는 접어두기

  예상 효과: 체감 정보량 50% 감소

  Phase 4 — 신뢰 요소 강화 (2~4주, 장기)

  가장 큰 효과, 가장 어려움.

  - 실제 베타 사용자 후기 1명 확보 → 가상 스토리 1개 교체
  - 통계(네이버 +27.4% 등) 출처 링크 추가
  - 대표·회사 정보 섹션 소폭 보강

  Phase 5 — 접근성·성능 (선택)

  - text-gray-400 → text-gray-500 이상 (WCAG AA 대비)
  - 동적 요소 aria-label 추가
  - 번들 사이즈 점검

  ---
  🗓 권장 실행 순서

  이번 주:     Phase 0 (측정) + Phase 1 (안전 정리)
  다음 주:     Phase 2 (페이지 분리)
  2~3주차:     Phase 3 (trial 분해)
  4주~:        Phase 4 (신뢰 요소) — 베타 후기 확보 시
  여유 시:     Phase 5 (접근성)
